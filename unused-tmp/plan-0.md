
# Текущий функционал

1. evo-root.sh - единая точка входа для команд CLI, project-evo-method
2. области: /meta - проверяет реализацию method, /method - реализация project-evo-method, /project--<name> - прикладной проект, ведомый по методике project-evo-method
3. обертка evo.sh - для запуска команд в контексте прикладного проекта
4. команда: check-document-metadata-header - проверить, что реализация /method подходит под правила document-metadata-header
5. команда: spec-to-code - сгенерить код по spec в проекте. Ограничения и особенности:
   - есть настройки, как вызвать LLM
   - в LLM подает промпт, в ответ получает текст
   - у LLM нет контекста, получает только промпт
   - для указания, в какие файлы записать части ответа, LLM размечает текст спец.форматом
   - спец.формат разметки текста для указания, как разместить его части по файлам - сейчас это собственный минимальный "file-operation block format". Поддерживается только action:create (в режиме rewrite) 
   - spec-to-code берет в работу только 1 файл spec (/spec/index.md)
