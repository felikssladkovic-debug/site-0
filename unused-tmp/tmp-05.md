# Структура файлов
Предлагаю такую структуру файлов для site-0 в самом-самом сокращенном виде:

## Общий вид

```
README.md
evo.sh

/meta - это Layer-0. Здесь инструменты
  /rules
    document-metadata-header.md
  /tools
    check-document-metadata-header.py
    /tools-tests
      /check-document-metadata-header
        test-1-required-headers.py
        test-2-no-other-headers.py
        test-3-ids-are-uniq.py

/method
  dummy-rule.md
  /tools
    spec-to-code.py
    /tools-tests
      /spec-to-code
        test-1-fails-if-no-project.py

/project--site-0
  /spec
    index.md
  /generated

```

## 1. README.md
README.md - это точка входа для человека, кто знакомится с project-evo-method.
Здесь указываем, что это исполняемая методика, и единая точка для всех CLI-команд - это evo.sh

## 2. evo.sh
evo.sh - через этот sh должны проходить все команды пользователя.
Там help - поэтому человек может не держать в памяти параметры запуска на все-все команды
Таким образом, выполнить spec-to-code это "evo.sh spec-to-code"
Сейчас нужно поддерживать такие 2 команды:
- check-document-metadata-header
- spec-to-code
Что делают эти команды - указано дальше по тексту.

## 3. /meta
/meta - это Layer-0.
Здесь инструменты для проверки, что корректно составлены файлы в /method.
Это нужно только на этапе, когда активно дорабатываем саму реализацию, среду project-evo-method.
Файлы в /meta - этим правилам могут не подчиняться.

### 3.1. /meta/rules
Здесь только текстовое описание правил - чтоб понимать, что вообще хотим. Команды evo.sh НЕ используют эти файлы, пока не указано прямо.

#### 3.1.1. /meta/rules/check-document-metadata-header.md
Здесь прописаны правила для оформления "document metadata header". Текст, который с тобой проработали.

### 3.2. /meta/tools
Здесь все, что касается инструментов, которые можем выполнять в рамках назначения /meta

#### 3.2.1. /meta/tools/check-document-metadata-header.py
Проверить, что в /method все файлы выполняют требования правила /meta/rules/check-document-metadata-header.md
Эта проверка - детерминированный код.
Этот код должен реализовать правила которые в md.
Но сам этот код файл .md не использует.
Проверки - это hard-code

Чтоб вызвать этот инструмент, нужно:
```bash
evo.sh check-document-metadata-header
```
##### 3.2.1.1. /meta/tools/tools-tests/
В этой папке тесты для инструментов из /meta/tools/. 
Так-как инструменты - это код, то для этого кода нужны тесты. Эти тесты - в этой папке.
Здесь используем правило именования - для инструмента <tool-name>.py код тестов находится в папке <tool-name>
Пример: /meta/tools/tools-tests/check-document-metadata-header


## 4. /method
/method - это Layer-1.
Здесь реализация для project-evo-method,
Это инструменты для проведения эволюции проектов. Это:
- программный код. Например, spec-to-code.py
- правила/промпты для LLM - они будут использованы в тех action, где выполнение проводит LLM
- правила/ограничения для ведения файлов/документов в прикладных проектах.
В том или ином виде - это настройки для выполнения action, предусмотренных project-evo-method.

### 4.1. /method/dummy-rule.md
Это (как и следует из названия) просто файл, без какого-либо полезного наполнения - нужен только, чтоб проверить работу утилит из /meta.

### 4.2. /method/tools/
Здесь все инструменты, доступные в project-evo-method.

#### 4.2.1. /method/tools/spec-to-code.py
Это spec-to-code.

#### 4.2.2. /method/tools/tools-tests/
Тесты для инструментов из /method/tools.
Так-как инструменты - это код, то для этого кода нужны тесты. Эти тесты - в этой папке.
Здесь используем правило именования - для инструмента <tool-name>.py код тестов находится в папке <tool-name>
Пример: /method/tools/tools-tests/spec-to-code


## 4. /project--site-0
Проект site-0, его workspace.

### 4.1. /project--site-0/spec
Папка для spec проекта.

#### 4.1.1. /project--site-0/spec/index.md
Единственный spec на данный момент.
Здесь не используем пока разбиение на impl-area. Вместо этого - просто 1 документ для spec, где и написано, что нужно.

## 4.2. /project--site-0/generated
Папка, в которую будет попадать сгенерированный код
